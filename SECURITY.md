# STRIX Security Policy

## Supported Versions

The latest main branch of STRIX is actively supported.

## Reporting a Vulnerability

Do not post security issues in public GitHub issues.

Report privately to maintainers with:

- Subject: Security Report - STRIX
- Affected component and version
- Reproduction steps
- Impact assessment
- Suggested remediation
- Sanitized proof of concept if available

## Response Targets

- Initial acknowledgement: within 72 hours
- Triage result: within 7 days
- Fix timeline: depends on severity and complexity

## Safe and Legal Use

This software must only be used against systems you own or have explicit written permission to test.

## Responsible Disclosure

- Report privately first
- Allow maintainers time to patch
- Coordinate public disclosure after fixes are available
